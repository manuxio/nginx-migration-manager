# Implementation Plan — nginx-managed

Goal: CSV → nginx reverse-proxy hosts, with a small Node/React UI. Secure nginx,
permissive hosts, hand-edits respected, no mass delete. Built for speed.

## Key decisions (chosen so we can move fast)

| Concern | Decision | Why |
|---|---|---|
| Reload trigger | File watcher in nginx container (`inotifywait` → `nginx -t && nginx -s reload`) | No Docker socket in the app = much smaller blast radius |
| Validate-before-reload | Watcher runs `nginx -t`; reload only on success, keep last-good + record error | A bad hand edit can't take the proxy down |
| Hand-edit handling | Marker-based field merge: only lines tagged `# managed:*` are rewritten; everything else preserved verbatim | "Respect lines you don't recognize, update the ones you do" |
| History / rollback | Git repo in the `proxy_conf` volume, auto-commit on apply | Free audit log + rollback + redundant edit detector |
| App auth | HTTP basic auth on :3000 | Cheap defense in depth |
| Delete | App never deletes; manual `rm` only | Required by spec |
| Cert | Single self-signed cert (SAN = configured domains + localhost) regenerated by entrypoint if missing | Works for all SNI hosts; one place to manage |
| HSTS | Off by default | Self-signed + HSTS = browser lockout |
| Unknown Host | Default server returns `444` | Only configured domains are served |

## Phases

### Phase 0 — Skeleton (15 min)
- `docker-compose.yml`: services `nginx` (80,443) and `app` (3000), three volumes
  (`proxy_conf`, `certs`, `app_data`), shared internal network.
- Empty `nginx/sites/` include wired into `nginx.conf`.

### Phase 1 — Hardened nginx (30 min)
- `nginx.conf`: `server_tokens off`, slowloris timeouts (`client_header_timeout`,
  `client_body_timeout`, `keepalive_timeout`, `send_timeout`), sane global
  `client_max_body_size`, `include /etc/nginx/sites/*.conf;`, default-deny server → `444`
  on both 80 and 443.
- `snippets/ssl.conf`: TLS 1.2+1.3, modern cipher list, `ssl_session_cache`, OCSP off
  (self-signed), HTTP/2 on.
- `snippets/proxy.conf` (permissive): `proxy_http_version 1.1`, `Upgrade`/`Connection`
  for websockets, forwarded headers, long read/send timeouts, `client_max_body_size 0`.
- `entrypoint.sh`: generate self-signed cert into `certs` volume if absent, launch
  `watcher.sh` in background, `exec nginx -g 'daemon off;'`.
- `watcher.sh`: debounce on `sites/` changes, `nginx -t` → reload, write
  `sites/.reload-status.json` (`{ok, time, error}`) for the UI.

### Phase 2 — Generator + manifest (45 min)
- `templates/proxy-host.conf.tmpl`: server block on 80+443, `proxy_pass
  http://<address>:<port>`, includes `ssl.conf` + `proxy.conf`, header comment.
- `server/csv.js`: parse `"domain","address","port"`, default port 80, validate
  (hostname/IP regex, port range), return rows + errors.
- `server/manifest.js`: load/save `manifest.json` (desired state for list/export).
- `server/nginxHost.js`: render template (with `# managed:*` markers), `mergeUpstream()`
  that surgically rewrites only marked lines, `parseUpstream()` for export, atomic write
  (tmp + rename). Implements create/update/unchanged/skip-manual/invalid from CLAUDE.md.
- `server/gitStore.js`: `git init` if needed, `commitAll(message)`, `log()`.

### Phase 3 — API (30 min)
- `POST /api/import` — multipart or text CSV; **dry-run by default**, returns per-domain
  plan (created/updated/skipped-locked/invalid); `?apply=true` to commit.
- `GET /api/hosts` — list with status + locked badge + last-known upstream.
- `GET /api/export` — current hosts back to CSV (round-trip).
- `POST /api/reload` — touch the watcher trigger; return last reload status.
- `GET /api/status` — read `sites/.reload-status.json`.

### Phase 4 — React UI (45 min)
- Hosts table: domain, upstream, status (managed/locked), reload state.
- CSV upload → shows the dry-run diff → "Apply" button.
- CSV export button.
- Inline reload-status banner (last `nginx -t` result).

### Phase 5 — Polish / nice-to-haves (pick from below as time allows)

## Easy features worth adding (ranked by value/effort)

1. **Dry-run diff before apply** — already in the API; cheap, prevents surprises. *(do this)*
2. **CSV export / round-trip** — list → CSV; lets you edit in a spreadsheet. *(do this)*
3. **Locked/manual badge** — surfaces which hosts the app won't touch. *(do this)*
4. **Per-host enable/disable** — rename `<domain>.conf` ↔ `<domain>.conf.disabled`
   (nginx ignores non-`.conf`). Pause a host without deleting. Cheap.
5. **Git-backed history** — `git init` the `proxy_conf` volume, auto-commit after every
   apply. Free audit log + one-command rollback. Also a second, very reliable way to spot
   hand edits (`git status` = dirty). ~20 min.
6. **Upstream health ping** — TCP/HTTP check each upstream, show up/down dots. ~20 min.
7. **Cert SAN regen button** — rebuild the self-signed cert to cover all current domains.
8. **CSV validation report** — show bad rows with reasons before import. (Comes free from
   `csv.js`.)
9. **Per-host extra-snippet field** — optional textarea appended into the host's
   `location` block for one-offs.

## Concerns / watch-outs

- **CSV is an injection vector.** The app turns CSV cells into filenames and nginx
  directives. Strictly validate domain (hostname/IP only) and reject path-traversal. This
  is the one place "app security doesn't matter" is wrong.
- **Self-signed cert** → browser warnings and no per-domain trust. Fine internally; plan a
  Let's Encrypt swap later (needs public DNS + port 80 reachable).
- **HSTS off** until real certs, else clients get permanently stuck on a cert they can't trust.
- **Don't publish :3000.** Even though the app is "network-protected," only map 80/443 to
  the host. Optionally add basic auth — it's ~5 lines.
- **Reload races.** Debounce the watcher so a burst of file writes triggers one reload.
- **Atomic writes.** Write to a temp file and `rename()` so nginx never reads a half-written conf.
- **Single cert + many domains.** SAN list can get long; regenerate on demand, not per request.
- **Binding 80/443** needs root or `cap_net_bind_service` — the official nginx image
  already handles this; don't fight it.
